/api/vendor/v2/webhooks get post

get

Returns an array hooks setup for this account

post

Adds a New Hook for this account When you create a new webhook and you want to have EireneRx send you a patient specific url or header. You will need to provide a string template so that EireneRx knows how to inject your patient specific identifier:

ex: { "endpoint": "http://example.com/<%= patient_id %>" ... }

In this example, the identifier will replace the <%= patient_id %> template string.

What if you want the identifier to come via a header?

You do the same thing but for the specified header:

ex:

{ ... "headers": { "X-Foo-Id": "<%= patient_id %>", 'X-Bar-Id": 111 } ... }

** Don't hesitate to contact us, if you have any questions. We are using a underscore templates to dynamically inject patient identifiers

/api/vendor/v2/patients get

get

When performing a get on the patients you can pass a query attribute called where, this attribute should have a value of a stringified json object:

In that JSON Object you have the option to query for 1 - 4 colums

  • first_name
  • last_name
  • dob (Date of Birth - YYYY-MM-DD )
  • ssn (Social Security Number - 555-55-5555 )

The result will return 1 to many patient json documents that meet the query criteria.

If you do not enter any query criteria, then the result will return an empty array.

Example: | curl -i -G -H 'X-Sponsor-Id: 153' --data-urlencode 'where={"ssn":"555-55-2032"}' https://testCognify:testCognify@staging-vendor-api.eirenerx.com/api/vendor/v2/ patients

/api/vendor/v2/patients/{identifier}/allergies put

put

Updates the list of allergies on a patient. The allergies sent from the vendor will be the source of truth.

  • allergies can be created with concept_id and concept_type.
  • if an allergy is not supplied with concept_id and concept_type, they will be defaulted to null and will not be clinically linked in eirenerx.

  • when patient list and sent in allergies list are compared:

    • allergies that are on patient list but not in the sent in list -> these allergies will be made inactive.
    • allergies that are not on the patient list but are in the the sent in list -> these allergies will be created.
    • if they are in both list -> allergies in EireneRx will be updated.

    Example body:

    {
  "allergies": [ 
    { 
      "allergy": "alprazolam", 
      "concept_id": "d00168", 
      "concept_type": "ingredient", 
      "onset_date": "01/01/1980", 
      "reaction": "shock", 
      "description": "allergic" 
    }, { 
      "allergy": "beesting" 
    }
  ]
}

    The only required field is allergy.

    • if no onset_date -> defaults to current date.
    • if no reaction -> defaults to empty string.
    • if no description -> defaults to allergy field
    • if no concept_type -> defaults to null
    • if no concept_id -> defaults to null
